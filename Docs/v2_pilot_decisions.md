## 2026-04-21 — `_default_grid` 5값 → 8값 (공통비 r 축소)

### 배경

Phase 6 e2e 결과 `min_topic_size=71` 선택 → 5 토픽 생성. v2.4 는 수작업으로 `mts=50` 을 박아 10 토픽 생성. sweep 생존자 `[37, 71, 137]` 중 median_low 가 71 을 골랐으나 실제로 **mts=37 (8 토픽)** 이 v2.4 결과에 근접한 후보였음.

근본 원인은 "UMAP 차원" 이 아니라 **grid 간격이 너무 넓어 sweet spot 영역 (≈50) 이 누락** + **median_low 가 중앙값을 선호**.

### 사용자 제안

> 공통비 r 줄이고 `[10, a, b, c, d==sqrt(N)/2, e, f, g]` 형태로 바꾸자.

즉 **8값 등비수열**, 하한 10, 중앙 `C = sqrt(N)/2` 를 index 4 로 고정.

### 도출 공식

```
grid[0] = 10,  grid[4] = C
→ r^4 = C/10
→ r = (C/10)^(1/4)
```

N=5590, C=37 → r = 3.7^0.25 ≈ **1.388** (기존 r ≈ 1.924 대비 더 촘촘)

### 새 grid (N=5590)

|---:|---:|---:|
| 0 | 10 | **10** |
| 1 | 10 × 1.388 = 13.88 | **14** |
| 2 | 10 × 1.388² = 19.26 | **19** |
| 3 | 10 × 1.388³ = 26.74 | **27** |
| 4 | C = 37 | **37** |

→ `[10, 14, 19, 27, 37, 51, 71, 99]`

### 현재 vs 새 grid 비교

| 항목 | 현재 (5값) | 새 (8값) |
|---|---|---|
| 공통비 r | 1.924 (거의 2배씩 점프) | 1.388 (1.4배씩) |
| 값들 | 10, 19, 37, 71, 137 | 10, 14, 19, 27, 37, 51, 71, 99 |
| 50 근처 | 없음 (37 → 71 공백) | **51 으로 커버** |
| 최대값 | 137 | 99 (감소) |
| 학습 횟수 | 5 | 8 (60% 증가) |

### 기대 효과

- **v2.4 sweet spot 50 근처 (51)** 가 grid 에 들어감 → 10 토픽 근처 결과 가능성
- 기존 생존자 37, 71 은 그대로 포함 → 직접 비교 가능
- tie-break 을 median_low 로 유지하면 8값 중 중앙 index 3~4 = **27 or 37** → 자연스럽게 세분화 선호
- 최대값 99 < 137 — "너무 굵은 클러스터" 영역 덜 탐색 (실무적으로 무의미했던 영역)

### N 스케일별 결과

| N | C | r | 새 grid |
|---:|---:|---:|---|
| 1,000 | 16 | 1.124 | [10, 11, 13, 14, 16, 18, 20, 23] |
| **5,590** | **37** | **1.388** | **[10, 14, 19, 27, 37, 51, 71, 99]** |
| 6,400 | 40 | 1.414 | [10, 14, 20, 28, 40, 57, 80, 113] |
| 10,000 | 50 | 1.495 | [10, 15, 22, 33, 50, 75, 112, 167] |
| 100,000 | 158 | 1.995 | [10, 20, 40, 79, 158, 315, 628, 1252] |

N=100k 같은 대규모 에서는 r ≈ 2 로 spread 다시 커짐 — 도메인 감각과 일치 (대규모 corpus 는 토픽 크기 폭이 큼).

### 구현 (`s3_cluster.py::_default_grid`)

```python
def _default_grid(N: int) -> list[int]:
    """8값 기하급수 그리드. center = sqrt(N)/2 (index 4)."""
    C = max(10, round(N**0.5 / 2))
    if C <= 10:
        return [10]
    r = (C / 10) ** (1 / 4)
    return [round(10 * r**i) for i in range(5)] + [round(C * r**i) for i in range(1, 4)]
```
