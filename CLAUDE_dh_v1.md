# CLAUDE_dh_v1.md — dh.kim 작업-방식 템플릿 (v1)

> **이 템플릿 쓰는 법**
> dh.kim의 재사용 가능한 **작업-방식 템플릿**이다. 새 프로젝트를 시작할 때 이 파일을 그 프로젝트 루트에
> `CLAUDE.md`로 복사하고, **§0 가드레일·§1 문서·§3 컨벤션의 프로젝트-특화 빈칸만 채워라.** 방법론(문서 먼저,
> 작업-단위 주도, 커밋 규율, 디렉터리 규약)은 그대로 둔다.
> 이 템플릿은 전역 `~/.claude/CLAUDE.md` **위에 얹히는 레이어**다 — 전역 규칙(단순성·외과적 수정·계획
> 먼저·검증 기준)을 **중복 기술하지 말고 전제**한 채, 프로젝트별로 더할 것만 적는다.

## 0. 프로젝트 가드레일 (불변, 최우선) — *프로젝트마다 채울 자리*

이 프로젝트의 **깨지면 안 되는 계약**을 여기 적는다. 리팩터·기능 추가 시 최우선으로 보존한다.
아래는 형식 예시일 뿐 — 프로젝트에 맞게 교체/삭제하고 실제 불변식만 남긴다.

- **시크릿 정책.** 시크릿은 환경변수만. config·코드·로그·산출물에 평문 기입·커밋 금지.
  *(예: `XXX_API_KEY`는 env 참조, config엔 빈 placeholder. "config 값 있으면 그것, 없으면 env" 패턴.)*
- **대용량/비추적 정책.** 재현 가능한 산출물·데이터·인덱스는 추적 금지(`.gitignore`). 코드·문서·소형 입력만 커밋.
  *(예: `outputs/`, `data/`, `*.bin` 제외 — 프로젝트별 목록을 여기 확정.)*
- **load-bearing 불변식.** 단계·모듈 간 통신 포맷, 파일명 규약, 외부 규약(라이선스·robots·rate-limit) 등
  **다운스트림이 조용히 의존하는 계약**을 한 줄씩 명시. *(예: "X 파일의 표 포맷은 Y가 regex로 재파싱하므로 고정.")*
- **조용한 누락 금지.** 행 drop, 필터 탈락, strip된 데이터 등은 반드시 로그/플래그로 명시한다.

> 가드레일은 1줄로 명료하게. "왜 깨지면 안 되는지(다운스트림 영향)"를 함께 적으면 미래의 자신이 안 깬다.

## 1. 문서 운영 (계획 먼저, 코드 나중)

| 파일 | 역할 | 갱신 시점 |
|---|---|---|
| `README.md` | **외부 시점 요약** (설치·사용·구조) | 스택/구조 변화 시 |
| `CLAUDE.md` (이 파일) | **작업 정책** | 규칙이 바뀌면 **여기 먼저** |
| `PLAN.md` (또는 `Docs/PLAN.md`) | **마스터 설계** (결정·단계·검증 기준) | 설계 결정이 바뀔 때 |
| `Docs/` | 설계·결정 로그·작업 노트·다이어그램 | 수시 |
| `TODO.md` | **Phase 진행 로그** (완료 + commit hash) | Phase 완료마다 |
| `Docs/KNOWN_LIMITATIONS.md` | 런타임 발견 이슈 추적 (append-only) | 발견·해결마다 |

**순서: 계획을 먼저 문서화 → 그 다음 구현.** 코드가 문서를 앞서지 않는다.
`KNOWN_LIMITATIONS.md`는 **append-only** — 해결 시 `## ✅ 해결됨` 표시, 취소된 항목은 삭제하지 말고
표시만 남긴다(provenance 보존). 규칙이 바뀌면 `CLAUDE.md`를 먼저 고치고 나서 코드를 고친다.

## 2. 작업 단위(이슈/Phase) 주도

실작업은 작업 단위로 끊어서 진행한다:

1. **단위 선언** — 작업 1개 = **목표 + 검증 기준(완료 조건)을 먼저 적는다** (`TODO.md`/`Docs/` 노트 또는 이슈).
2. **커밋** — 작업하며 커밋, 메시지에 작업 단위를 참조한다.
3. **검증 후 완료** — 검증 기준 충족을 확인하고 `TODO.md` 진행 로그에 결과 + commit hash를 남긴다.

규율: **한 Phase = 한 클린 커밋, 단계 건너뛰지 않기, 진행 전 기준 검증.**
Git remote(GitHub 등) 연결 시 작업 단위를 **이슈로 승격**하고 `refs #N`(진행) / `Closes #N`(완료) 규약을 더한다.
가능하면 **이슈 번호 ↔ 디렉터리 서브번호**를 맞춘다(이슈 #21 ↔ `02_xxx/021_*`).
정책·문서 부트스트랩(이 `CLAUDE.md` 등)은 이 규칙의 예외다.

## 3. 디렉터리 컨벤션

- **최상위 = 파이프라인/작업 단계**, `NN_이름` prefix.
- **하위 = 서브스텝/이슈 단위**, `NNM_이름` (상위 번호 + 일련).
- 루트 직속: `CLAUDE.md`, `README.md`, `PLAN.md`, `Docs/`, `.gitignore` — **단계 폴더 밖**.
- 단계 폴더는 **해당 작업 착수 시 생성**(이슈와 함께). **빈 폴더를 미리 만들지 않는다.**
- 폐기·구버전 자료(`Archive_*`, `depricated/` 등)는 **보존하되 손대지 않는다**(참고용).
- 새 메이저 버전은 정제 복사본을 새 `NN_..._verX` 폴더로 만든다(기존 폴더는 그대로 둠).

```
01_setup/            # 환경·의존성 고정 (Phase 0)
02_xxx/              # 작업 단계 (Phase 1)
  021_subtask_a/     #   서브스텝/이슈 단위
  022_subtask_b/
03_yyy/              # 작업 단계 (Phase 2)
04_zzz/              # 작업 단계 (Phase 3)
Docs/                # PLAN.md, 결정 로그, 다이어그램
Archive_old/         # 폐기 자료 보존 (손대지 않음)
CLAUDE.md  README.md  PLAN.md  .gitignore   # 루트 직속
```

## 4. 커밋 규칙

- 한 커밋 = **한 논리 변경**. **버그픽스 / 리팩터 / 새 기능 / 문서는 섞지 말고 쪼개서** 커밋한다.
- 커밋 메시지 끝에 `Co-Authored-By: Claude ...` 라인.
- 초기엔 `main`에 직접. **협업/리뷰가 필요해지면 그때 브랜치**를 도입한다.
- Phase 완료 커밋은 `TODO.md` 로그에 hash를 남긴다. 가능하면 커밋에서 이슈를 참조한다.

## 5. 코드 작업 원칙 (generic)

전역 `~/.claude/CLAUDE.md`(단순성 우선·외과적 수정·검증 기준)를 **전제**하고, 여기선 강조점만:

- **외과적 수정.** 요청에 직접 닿는 줄만 바꾼다. 인접 코드·주석·포맷을 "개선"하지 않는다.
- **기존 동작 보존 + 신규는 opt-in.** 기본 동작은 byte-identical하게 유지하고, 새 기능은 플래그/옵션으로 켠다.
- **단순성 우선.** 요청 범위 밖 추상화·"유연성"·불가능 시나리오 방어 코드는 넣지 않는다.
- **중복 발견 시 통합.** 같은 로직이 두 곳에 있으면 손대는 김에 한 곳으로 모은다(건드릴 때만).

> 프로젝트별 코드 컨벤션(모듈 계약·config 레이어링·로깅·언어 규약 등)이 있으면 이 절 아래에 추가한다.
